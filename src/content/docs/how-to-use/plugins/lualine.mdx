# Lualine

[lualine.nvim](https://github.com/nvim-lualine/lualine.nvim) is a modern status line plugin for Neovim that provides a clean, customizable status line with various components and themes.

## Key Features

- Modern and clean design
- Multiple themes
- Customizable components
- Git integration
- LSP status
- File type icons
- Performance optimized
- Easy configuration

## Basic Usage

### Default Components

- Mode indicator
- Git branch
- File name
- File type
- Line/column numbers
- LSP status
- Search count
- Location/percentage

### Sections

- Left section: Mode, Git, File info
- Middle section: Diagnostics, Search
- Right section: LSP, Line/Column, Location

## Configuration

### Basic Setup

```lua
require('lualine').setup({
  options = {
    icons_enabled = true,
    theme = 'auto',
    component_separators = { left = '', right = ''},
    section_separators = { left = '', right = ''},
    disabled_filetypes = {
      statusline = {},
      winbar = {},
    },
    ignore_focus = {},
    always_divide_middle = true,
    globalstatus = false,
    refresh = {
      statusline = 1000,
      tabline = 1000,
      winbar = 1000,
    }
  },
  sections = {
    lualine_a = {'mode'},
    lualine_b = {'branch', 'diff', 'diagnostics'},
    lualine_c = {'filename'},
    lualine_x = {'encoding', 'fileformat', 'filetype'},
    lualine_y = {'progress'},
    lualine_z = {'location'}
  },
  inactive_sections = {
    lualine_a = {},
    lualine_b = {},
    lualine_c = {'filename'},
    lualine_x = {'location'},
    lualine_y = {},
    lualine_z = {}
  },
  tabline = {},
  winbar = {},
  inactive_winbar = {},
  extensions = {'nvim-tree'}
})
```

### Custom Theme

Create a custom theme:

```lua
local custom_theme = {
  normal = {
    a = { fg = '#ffffff', bg = '#000000' },
    b = { fg = '#ffffff', bg = '#000000' },
    c = { fg = '#ffffff', bg = '#000000' },
  },
  insert = {
    a = { fg = '#ffffff', bg = '#000000' },
    b = { fg = '#ffffff', bg = '#000000' },
    c = { fg = '#ffffff', bg = '#000000' },
  },
  visual = {
    a = { fg = '#ffffff', bg = '#000000' },
    b = { fg = '#ffffff', bg = '#000000' },
    c = { fg = '#ffffff', bg = '#000000' },
  },
  replace = {
    a = { fg = '#ffffff', bg = '#000000' },
    b = { fg = '#ffffff', bg = '#000000' },
    c = { fg = '#ffffff', bg = '#000000' },
  },
}

require('lualine').setup({
  options = {
    theme = custom_theme,
  },
})
```

### Custom Components

Add custom components:

```lua
local function get_python_venv()
  local venv = os.getenv('VIRTUAL_ENV')
  if venv then
    return string.match(venv, '([^/]+)$')
  end
  return ''
end

require('lualine').setup({
  sections = {
    lualine_a = {'mode'},
    lualine_b = {'branch', 'diff', 'diagnostics'},
    lualine_c = {
      {'filename', path = 1},
      {get_python_venv, icon = 'üêç'},
    },
    lualine_x = {'encoding', 'fileformat', 'filetype'},
    lualine_y = {'progress'},
    lualine_z = {'location'}
  },
})
```

## Advanced Features

### Component Options

Configure component options:

```lua
require('lualine').setup({
  sections = {
    lualine_c = {
      {
        'filename',
        path = 1,
        symbols = {
          modified = '[+]',
          readonly = '[-]',
          unnamed = '[No Name]',
          newfile = '[New]',
        },
      },
    },
  },
})
```

### Conditional Components

Add conditional components:

```lua
local function get_lsp_client()
  local clients = vim.lsp.get_active_clients()
  if #clients > 0 then
    return clients[1].name
  end
  return ''
end

require('lualine').setup({
  sections = {
    lualine_x = {
      {get_lsp_client, icon = 'üîç'},
      'encoding',
      'fileformat',
      'filetype',
    },
  },
})
```

### Extensions

Enable extensions for additional features:

```lua
require('lualine').setup({
  extensions = {
    'nvim-tree',
    'toggleterm',
    'quickfix',
    'fugitive',
  },
})
```

## Tips and Tricks

1. **Performance**

   - Use minimal components for better performance
   - Disable unused extensions
   - Set appropriate refresh rates

2. **Customization**

   - Create custom components for specific needs
   - Use conditional components for dynamic content
   - Customize themes to match your colorscheme

3. **Integration**
   - Enable relevant extensions
   - Configure component separators
   - Set up global status line when needed
